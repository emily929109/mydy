# News Data Standardization Skill

此 Skill 用於規範將 18篇文章格式標準化，將原為18份獨立檔案轉化為 JSON 格式的流程(已建置)。

## 版本: v1.2

## 資料結構定義 (Schema)

每篇文章應包含以下欄位：

| 欄位名稱 | 類型   | 說明                                           |
| :------- | :----- | :--------------------------------------------- |
| `id`     | String | 文章唯一識別碼（建議使用原始檔名）。           |
| `banner` | String | 文章頂部橫幅圖片路徑（統一使用 `/` 開頭）。    |
| `title`  | String | 文章主標題。                                   |
| `txt`    | String | 內容主體，支援簡化 Markdown 與基礎 HTML 標籤。 |

## `txt` 內容規範

為了保持結構簡潔，`txt` 欄位應遵循以下標記規則：

1. **子標題**：使用 `|` 符號開頭，並加上粗體、字體大小為1.1rem。
   -- 請將原本使用 `#` 或 `*` 符號開頭者文字加上粗體、字體大小為1.1rem。
2. **條列清單**：使用 `．` 或 `1.`符號。
3. **圖片嵌入**：使用 `<img src="..." />` 標籤，路徑應為相對根目錄的路徑。
4. **重點強調**：使用 `**文本**` 或 `<strong>` 或行內樣式 ( 例如 `style="color:red"`)。
5. **排除內容**：忽略原始 HTML 中的 `class`、`style`、`id` 等屬性，以及特定的 UI 框架標籤（如 `row`, `col`, `fs-1` 等）。
6. **自定義樣式** : 行與行需保持一定的間距，可寫在行內樣式中，文本字體大小為1rem
7. **遇到連結** : 例如文章底部有立即申辦、立即購買、立即了解申辦流程等連結，請參考以下結構 `<p style=\"margin-bottom: 1.5rem; font-size: 1rem;\"><a href=\"/Home/Register\">立即申辦</a></p>`

## 轉換工作流

1. **提取 Banner**：從原始檔案中尋找 `img` 標籤且 `width` 或 `style` 顯示為 100% 的頂部大圖。
2. **提取 Title**：尋找 `h2` 標籤或具備 `fw-bold` 類別的段落。
3. **清洗內容**：
   - 將所有 `h3` 標題轉換為 `# `。
   - 將 `ul/li` 轉換為條列式文字。
   - 保留重要圖片連結，並將 `~/img/` 修正為 `/img/`。
   - 將純文字段落合併，並以換行符 `\n` 分隔。
4. **驗證格式**：確保產出的 JSON 符合 RFC 8259 規範，且編碼為 UTF-8。
5. **多圖處理**：若文章內有多張連續圖片，建議在 `txt` 中依序排列。
